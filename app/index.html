// About the project

Infrastructure as Code, delivered as a pipeline.

AutoDeploy automates the full provisioning and deployment lifecycle — turning every commit into a reproducible, observable, production-ready release on AWS.

A complete DevOps loop

From source to running infrastructure — every step is codified, reviewed, and observable. AutoDeploy combines IaC, CI/CD, and monitoring into one cohesive workflow.

  • ▸ Infrastructure as Code — Terraform modules describe VPCs, EC2, and security groups declaratively.
  • ▸ CI/CD pipeline — GitHub Actions runs plan, apply, and post-deploy checks on every push.
  • ▸ Monitoring — Prometheus scrapes metrics, Grafana visualizes them, Node Exporter watches the hosts.
  • ▸ Operational scripts — Shell automation for deploy, rollback, backup, and health checks.

// Folder structure

Clean, modular, and easy to navigate.

Code is organized by concern — Terraform modules, environments, monitoring, and automation each live in their own space.

autodeploy/
├── terraform/
│   ├── modules/
│   │   ├── ec2/           # EC2 instance module
│   │   └── security-groups/
│   └── environments/
│       └── dev/           # dev environment
│           ├── main.tf
│           ├── variables.tf
│           └── outputs.tf
├── .github/
│   └── workflows/
│       └── deploy.yml     # GitHub Actions CI/CD
├── monitoring/
│   ├── prometheus/
│   │   └── prometheus.yml
│   └── grafana/
│       └── dashboards/
├── scripts/
│   ├── deploy.sh
│   ├── rollback.sh
│   ├── backup.sh
│   └── health-check.sh
├── README.md
└── LICENSE

Modular by design. Each top-level directory has a single responsibility — infrastructure, pipelines, observability, or automation.

Environments are isolated under terraform/environments/, so dev, staging, and prod evolve independently without coupling.

terraform/modules environments/dev .github/workflows monitoring/prometheus monitoring/grafana scripts/*.sh
// Observability

Live monitoring, end to end.

Every deployed environment is observable from the moment Terraform finishes applying — metrics, dashboards, and host telemetry are wired in by default.

Prometheus

Pulls metrics from app and infrastructure targets on a fixed scrape interval, with alerting rules for SLO violations.

Grafana

Pre-built dashboards visualize CPU, memory, request rate, and deployment health across environments.

Node Exporter

Host-level metrics — CPU, memory, disk, and network — exposed for Prometheus to scrape from every EC2 node.

Ship infrastructure like software.
